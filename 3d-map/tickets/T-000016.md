# T-000016: Monochrome UI Redesign with Custom Zoom Controls

## Description
Replace Cesium's default UI containers with custom-built, monochrome aesthetic controls. Implement open-source zoom controls library and redesign the entire UI (panels, buttons, dropdowns) to follow a cohesive monochrome (grayscale + one accent color) design language.

## Requirements
1. **Research & Select UI Library**: Evaluate open-source libraries for custom controls
   - Options to explore:
     - Bootstrap Icons + vanilla CSS (lightweight, no framework)
     - Tailwind CSS (utility-first, monochrome compatible)
     - shadcn/ui (headless, composable)
     - Material Design Icons (customizable, monochrome)
   - Decision: Recommend lightweight approach to avoid bloat with Cesium

2. **Implement Custom Zoom Controls**
   - Replace Cesium's default zoom buttons (top-right corner)
   - Create custom +/- buttons with better styling
   - Add zoom-in/zoom-out keyboard shortcuts (+ / - keys)
   - Display current zoom level / altitude

3. **Redesign UI Panels**
   - Info Panel (top-left): Monochrome cards, proper spacing
   - Controls Panel (top-right): Organized sections with clear hierarchy
   - Use consistent color palette: blacks, whites, grays + 1 accent (suggest: cyan/blue)
   - Ensure responsive design (mobile-friendly)

4. **Typography & Spacing**
   - Clean, minimal fonts (system fonts preferred)
   - Consistent padding/margins following a grid system
   - Icon set integration (suggest: Material Icons or Feather Icons)

## Acceptance Criteria
- [ ] Custom zoom controls implemented and functional
- [ ] All UI panels redesigned with monochrome aesthetic
- [ ] No visual regression: all existing buttons/inputs work
- [ ] Responsive on desktop (mobile nice-to-have)
- [ ] CSS is organized and maintainable
- [ ] Library choice documented in code comments
