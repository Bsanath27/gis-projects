# T-000017: Fix Pincode Search & Boundary Highlighting

## Description
Debug and fix the pincode boundary highlighting feature. Currently, when users enter a pincode and click "Highlight Pincode", the boundaries don't change color. Root cause is a race condition in property value retrieval from GeoJSON features.

## Root Cause Analysis
In `src/utils/pincodeDataSource.js` lines 50-56:
- Property access uses `.getValue?.()` which may fail during GeoJSON loading
- Type mismatch between user input (string) and stored property values
- Race condition: properties not immediately available after GeoJSON load
- No logging/debugging to identify which properties are available

## Requirements
1. **Debug Pincode Properties**
   - Add console logging in `highlightPincode()` to show:
     - Total entities in dataSource
     - All properties for first 3 entities
     - Pincode values found and their types
   - Identify if properties are consistent across all features

2. **Fix Property Value Retrieval**
   - Test both `.getValue()` and direct property access
   - Normalize both search value and property values (trim, lowercase, convert to string)
   - Handle missing/undefined pincode properties gracefully

3. **Add Input Validation**
   - Validate pincode format (6 digits)
   - Provide user feedback (success message or "pincode not found")
   - Case-insensitive search

4. **Implement Search Enhancements**
   - Add fuzzy search by office name (e.g., "Sahupuram" → highlight matching pincodes)
   - Add district-based filtering (optional nice-to-have)

## Testing
- Test with known pincodes from data (628229, 600001, etc.)
- Test with invalid pincodes (should handle gracefully)
- Test boundary color change (CYAN highlight)
- Verify boundaries snap to terrain correctly

## Acceptance Criteria
- [ ] Pincode highlighting works reliably for valid pincodes
- [ ] Boundaries change to CYAN color when highlighted
- [ ] User gets feedback if pincode not found
- [ ] Console shows clear debug logs
- [ ] Office name search implemented
