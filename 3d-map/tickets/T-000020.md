# T-000020: WebGL Overlay with Three.js Integration

## Description
Integrate Three.js as a WebGL overlay on top of Cesium.js to enable advanced graphics capabilities including particle effects, custom geometries, complex shaders, and advanced post-processing. This allows extending beyond Cesium's native capabilities while maintaining full terrain and pincode visualization.

## Feasibility Analysis
✅ **FEASIBLE** - Three.js can be overlaid as a separate canvas on top of Cesium using:
1. Canvas overlay approach (transparent Three.js canvas above Cesium canvas)
2. Event delegation system for mouse/keyboard coordination
3. Coordinate transformation layer between ECEF (Cesium) and Cartesian (Three.js)

## Technical Approach

### Architecture
```
┌─────────────────────────────────────────┐
│  Three.js Overlay Canvas (transparent)   │ ← Custom effects, particles
├─────────────────────────────────────────┤
│  Cesium Canvas (main viewer)             │ ← Terrain, pincodes, buildings
└─────────────────────────────────────────┘
```

### Coordinate Synchronization
- Manual transformation from Cesium ECEF (Earth-Centered Earth-Fixed) to Three.js Cartesian
- Camera sync: Cesium camera changes → Three.js camera updates
- Real-time position updates using frame callbacks

### Render Loop
```javascript
// Synchronized render cycle
cesiumViewer.scene.preRender.addEventListener(() => {
  syncThreeJsCamera();        // Update Three.js camera from Cesium
  threeRenderer.render(...);  // Render Three.js overlay
});
```

## Requirements

### Phase 1: Foundation Setup (BLOCKING for other features)
1. **Dependencies Integration**
   - Add Three.js (^r154) to package.json
   - ~300KB gzipped addition
   - Integration with Vite build system

2. **Canvas Overlay System**
   - Create transparent canvas overlaid on Cesium container
   - CSS positioning and z-index management
   - Resize handling (responsive)

3. **Coordinate Transformation Layer** ⚠️ CRITICAL
   - Build ECEF ↔ Cartesian conversion utilities
   - Test with known coordinates (Himalayas, pincodes, etc.)
   - Handle edge cases (poles, date line)

4. **Camera Synchronization**
   - Hook into Cesium camera.changed event
   - Update Three.js orthographic/perspective camera
   - Maintain aspect ratio and frustum

5. **Event Delegation System**
   - Coordinate mouse events between Two canvases
   - Click detection: prioritize Three.js if object hit, else Cesium
   - Touch event handling for mobile

### Phase 2: Advanced Features (Optional, depends on use cases)
1. **Custom Particle System**
   - Weather effects (rain, snow)
   - Building dust/debris
   - Energy visualization

2. **Advanced Shaders**
   - Custom lighting models
   - Heat map visualization
   - Texture distortion effects

3. **Custom Geometry**
   - Procedural building deformation
   - Terrain modification overlay
   - Vector field visualization

- **Memory Overhead:** +100-150 MB (Three.js + textures)
- **FPS Impact:** Maintain 55+ FPS on modern hardware
- **Load Time:** +0.5-1s additional startup time

## Known Challenges & Mitigations

| Challenge | Impact | Mitigation |
|-----------|--------|-----------|
| **Coordinate Transform Complexity** | High | Pre-build utility functions, extensive testing |
| **Render Loop Coordination** | High | Use Cesium preRender hook for synchronization |
| **Event Routing** | Medium | Implement raycaster for Three.js hit detection |
| **Memory Overhead** | Medium | Use WebGL texture atlasing, object pooling |
| **Browser Compatibility** | Low | Test on Chrome, Firefox, Safari (WebGL2 required) |

## Alternative Solutions Considered

| Option | Pros | Cons | Decision |
|--------|------|------|----------|
| **Babylon.js + Cesium** | Official integration, better docs | Requires migration | ❌ Too late to switch |
| **Cesium PostProcessStage** | Lightweight, native | Post-processing only, can't add 3D objects | ⚠️ Use for shaders |
| **Canvas.getContext('webgl')** | Full control | Too low-level, reinvent wheel | ❌ Not practical |
| **Three.js Only** | Simplest | Lose terrain/Cesium benefits | ❌ No |
| **Three.js Overlay** | Best of both worlds | Coordinate complexity | ✅ Selected |
